【プロジェクト名】
K-nine：AIPC向け対話型ファイルエクスプローラ with RAG

【概要】
このプロジェクトでは、RAG（検索拡張生成）技術とファイルエクスプローラ機能を組み合わせた、チャットベースのローカルAIファイル管理アプリを開発する。
ユーザーは自然言語でファイルを検索・要約・整理でき、煩雑なフォルダ操作や手動検索から解放される。すべての処理はローカルで完結し、セキュアかつ高速なファイル操作体験を実現する。

【主な特徴・機能】
- 🔍 ファイル検索：「◯◯に関する資料を探して」「2023年の請求書を見せて」などの自然文で検索可能（RAGで実装）
- 🧠 内容要約：文書ファイル（PDF, Word, txtなど）の中身をチャットで要約・キーワード抽出・構造化
- 🗂️ フォルダナビゲーション：「〇〇フォルダを開いて」「最近使ったファイルを一覧で」などの指示に対応
- 💬 チャットUIでの操作：すべての操作をチャット形式で実施（GUI不要）
- 📎 ファイルプレビュー連携：チャット内にファイルサムネイルや一部内容を表示
- 🔐 完全ローカル処理：オフライン対応。AIPC上のローカルLLM + ベクトルDBで構築

【使用技術スタック（想定）】
- LLM：Phi-3, Llama 3, Gemmaなど（AIPC搭載）
- ベクトルDB：ChromaDB or FAISS
- フロントエンド：Streamlit, Electron, または Flask + Web UI
- バックエンド：LangChain または LlamaIndex による RAG
- 文書解析：PyMuPDF, python-docx, pdfplumber など

【UX体験のまとめ】

1. ユーザーがどのような時に使うか
- 議事録・資料を探したいとき
- 途中から参加したプロジェクトを把握したいとき
- 記憶が曖昧な場合でも情報にアクセスしたいとき
- 複数ファイルを比較したいとき
- 定期的な作業を効率化したいとき

2. 操作の流れ
◆ ファイル検索
- 自然文で検索 → 候補と要約が表示 → シングルクリックで要約プレビュー → ダブルクリックで開く

◆ ファイル要約
- ファイル指定または検索から選択 → 「要約して」指示 → 要約表示

◆ チャット形式の利点
- 「この部分を詳しく」「2023年以降のPDFだけ」など細かい指示が可能

3. ユーザーの感じるベネフィット
- ストレスのない検索と要約
- GUI不要で誰でも使える
- 未知のファイルも自然文で発見可能
- セキュアなオフライン動作
